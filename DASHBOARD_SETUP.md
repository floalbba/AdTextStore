# Личный кабинет AdTextStore — настройка и проверка

## Переменные окружения

В `.env` должны быть заданы:

- `DATABASE_URL` — строка подключения к PostgreSQL (Neon)
- `DIRECT_URL` — прямое подключение (для миграций)
- `AUTH_SECRET` или `NEXTAUTH_SECRET` — секрет для NextAuth
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` — для входа через Google

## Миграция базы данных

Если схема уже применена через `prisma db push`:

```powershell
npx prisma generate
```

Для чистой миграции (с нуля):

```powershell
npx prisma migrate deploy
```

Если миграции в неконсистентном состоянии:

```powershell
npx prisma db push --accept-data-loss
```

**Если таблица Like отсутствует** (ошибка на /dashboard/public):

```powershell
npm run db:like
```

Или вручную: Neon Console → SQL Editor → выполнить `prisma/migrations/manual-add-like.sql`

## Запуск

```powershell
npm run dev
```

Откройте http://localhost:3000, войдите через Google и перейдите в личный кабинет.

## Проверка функциональности

1. **Создание промта**  
   - На главной странице кабинета нажмите «+ Новый промт»  
   - Заполните заголовок и текст, включите/выключите «Публичный»  
   - Нажмите «Создать» — карточка должна появиться в списке  

2. **Редактирование**  
   - На карточке нажмите иконку карандаша  
   - Измените данные и сохраните  

3. **Удаление**  
   - Нажмите иконку корзины  
   - Подтвердите удаление  

4. **Публичный/приватный**  
   - Нажмите иконку Globe/Lock на карточке  
   - Промт должен переключаться между публичным и приватным  

5. **Избранное**  
   - Нажмите звёздочку на карточке  
   - Промт должен появиться в разделе «Избранное»  

6. **Публичные промты**  
   - Перейдите в «Публичные промты» в сайдбаре  
   - Должны отображаться только промты с `isPublic = true`
